## 1. 仅增删改现有场景的音乐曲目

- 直接去对应的音乐文件夹（如 `Music/Default/Biomes/OverworldDay/`）增加、替换或删除音乐文件即可。（注：请不要放置MP3格式，否则会使曲目切换功能失效）
- 无需修改任何代码。



## 2. 直接替换某个原版音乐（全局/条件替换）

**操作**：

1. 在 `Core/MusicScenes/` 下新建一个分类目录以及一个类，继承 `MusicScene`
2. 在 `Core/MusicPath.cs` 里新建一个资源路径
3. 在对应的 Config 里新增加一个配置项目控制开关
4. 重写：
   - `Path` → 指向你刚建立的路径（如 `MusicPath.Default.Biomes.XXX`）
   - `IsEnabled` → 绑定配置开关
   - `ReplaceMusicId` → 填要替换的原版 `MusicID.XXX`
   - `ReplaceCondition` → 全局替换直接 `return true`，否则输入条件

   - `Priority` → 确保**不小于**你想覆盖的场景的优先级
   - `Condition` → 写你的触发条件（如 `player.ZoneXXX && Main.dayTime`）
   - `IsOtherworldly` → 如果想要设置异界版音乐，请将此设为 `True`



## 4. 注意事项

- `Default` 表示原版音乐，`Otherworldly ` 代表异界版音乐，请注意播放场景。
- 如果只有播放场景而没有对应的音频资源，则触发原版/默认音乐。
- 曲目资源**请不要使用mp3格式**
- **直接替换的优先度高于其他方式**，请确保你使用的方式正确。
- **当前架构不对音乐盒进行更改/替换**，如有需要欢迎自行添加音乐盒/拓展已有算法。
